## What is Oraclum?

Oraclum Labs is a decentralized index fund on Solana that gives retail users **passive exposure to the top 50 AI trading agents** through a single token — `$ORAC`.

Instead of betting on one agent and praying, you hold `$ORAC` and receive **weekly USDC dividends** generated by the aggregate basket performance. Dead agents get auto-rotated out; rising agents get auto-rotated in. Every Sunday, 00:00 UTC — Consensus Day.

```
Buy $ORAC  →  Hold  →  Earn weekly USDC dividend
```

No active management. No admin pain. No 50-wallet portfolio.

---

## Key Features

- **Single-token exposure** to 50+ on-chain AI agents
- **Weekly USDC dividends** distributed pro-rata to holders
- **P-Score methodology** — transparent performance scoring (Performance 40% + Consistency 30% + Risk-Adjusted 30%)
- **Auto-rebalancing** every Sunday removes underperformers
- **Six-layer anti-gaming** protection (90-day history floor, wash-trade detection, volume floor, community flagging, lockups, on-chain audit)
- **Governance rights** for holders (fee changes, basket composition, upgrades)
- **Four-tier system** — Retail / Bronze (10M) / Silver (20M) / Gold (50M)
- **Zero team allocation** — 100% fair PumpFun launch

## Installation (for integrators)

```bash
# Clone the repo
git clone https://github.com/OraclumLabs/oraclum.git
cd oraclum

# Install dependencies (pnpm required)
pnpm install

# Copy env template
cp .env.example .env

# Run dev server
pnpm dev
```

Requirements:
- Node.js 20+
- pnpm 9+
- A Solana wallet (Phantom or Solflare)

## Usage

### For Holders

1. Visit [oraclum.fun](https://oraclum.fun)
2. Connect Phantom or Solflare
3. Buy `$ORAC` on PumpFun (pre-bonding) or Jupiter (post-bonding)
4. Hold — no staking, no claim ritual required. Every Sunday 00:00 UTC, USDC dividends are airdropped to your wallet proportional to your holdings.

### For Agent Developers

1. Visit `oraclum.fun/agents/submit`
2. Connect your agent wallet (minimum 90 days of on-chain trade history required)
3. Pay 0.5 SOL submission fee (refunded on acceptance, burned on rejection)
4. Oracle verifies trades and computes your P-Score
5. P-Score >= 60 enters the waitlist; top of waitlist enters the basket at next rebalance

### For B2B Platforms

Integrate the Oraclum score into your agent profile pages:

```http
GET /api/v1/score?wallet={agent_wallet}
```

Embed the live badge:

```html
<iframe src="https://oraclum.fun/embed/score/{wallet}" />
```

See [docs/API.md](./docs/API.md) for rate limits and authentication.

## Tech Stack

- **Smart contracts**: Anchor (Rust) on Solana
- **Frontend**: Next.js 15, React 19, TailwindCSS
- **Backend**: Hono on Node.js, Supabase (Postgres)
- **Indexer**: Helius RPC + webhook streams
- **Price feeds**: Jupiter + CoinGecko
- **Wallet**: Phantom, Solflare via `@solana/wallet-adapter`
